## System Architecture

The pipeline has four stages, each handled by a different layer of the stack:

### Stage 1 — Natural Language Brief

The operator writes a one-liner like: *"30-second product film for SloerSwarm showing multi-agent orchestration with role-aware operators and mission directives."*

This becomes the input to Claude 4.6 Sonnet via the `@ai-sdk/anthropic` provider.

### Stage 2 — AI Code Generation

Claude receives the brief along with a Remotion skills context that includes:

- Available scene components (`TitleScene`, `StatsGridScene`, `FeatureListScene`, etc.)
- The `productVideoConfigs.ts` schema for structured metadata
- Animation utilities (`interpolate`, `spring`, `useCurrentFrame`)
- Brand constraints (colors, typography, timing)

The model outputs a complete `*.tsx` composition file — valid React, valid Remotion.

### Stage 3 — Preview and Iteration

The generated composition is hot-reloaded in the browser via Remotion's `<Player />` component. The operator can:

- Scrub through the timeline
- Adjust props in real-time
- Request revisions from Claude with follow-up prompts
- Compare multiple takes side-by-side

### Stage 4 — Render and Publish

Once approved, `@remotion/renderer` exports the composition to MP4. The file is deployed to the CDN and embedded on the product page via `ProductMarketingShowcase`.

Total time from brief to published: **under 15 minutes**.

## The AI Agent Loop

The key insight is that this is not a one-shot generation. It is an **agentic loop**:

1. Claude generates code
2. The system validates it compiles
3. If errors exist, Claude receives the error output and self-corrects
4. The cycle repeats until the composition renders cleanly

This is Multi-Chain Prompting in practice — the LLM orchestrates a pipeline where it is both the author and the debugger.

## What Makes This Different from Runway or Sora

Generative video tools like Runway Gen-4 and OpenAI Sora produce visually stunning clips. But they are **stochastic** — every generation is unique, and fine-grained control is limited.

Remotion plus Claude gives you:

- **Deterministic output** — Same code, same video, every time
- **Pixel-perfect control** — Every frame is computed, not generated
- **Version control** — The video lives in Git alongside your codebase
- **Composability** — Build scenes once, reuse across products
- **Type safety** — TypeScript catches errors before rendering

The tradeoff is clear: Runway for artistic exploration, Remotion for production-grade infrastructure.

## Lessons Learned

1. **Constrain the output space** — Give Claude specific components and brand rules. Unconstrained generation produces creative but inconsistent results.
2. **Validate at every step** — Run TypeScript compilation before rendering. Catch errors early.
3. **Use skills, not raw prompts** — Remotion's Claude skills package provides domain context that dramatically improves output quality.
4. **Keep humans in the loop** — AI generates the first draft. Humans approve the final cut.
